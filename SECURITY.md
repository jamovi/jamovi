# Security Policy

This policy covers the jamovi desktop application.

jamovi is developed and maintained by a small open-source team. Support is provided informally via email and GitHub, rather than a formal helpdesk with tracked tickets or SLAs — security issues are treated as a priority and fixed aggressively.

## Supported Versions

jamovi is released on two channels: **solid**, a more thoroughly tested line intended for users who want stability (comparable in spirit to a long-term-support release), and **current**, which carries the latest features. Security fixes are addressed on both channels. Users should keep jamovi up to date to receive security fixes.

## Reporting a Vulnerability

If you believe you've found a security vulnerability in jamovi, please report it privately rather than opening a public issue:

- email [contact@jamovi.org](mailto:contact@jamovi.org), or
- use [GitHub's private vulnerability reporting](https://github.com/jamovi/jamovi/security/advisories/new) for this repository

Please include a description of the issue, the steps to reproduce it, and the version of jamovi affected. Confirmed vulnerabilities are treated as a priority — we typically release a fixed version within 24 hours.

Where a reported vulnerability affects a third-party dependency, we will report it to the relevant upstream maintainer in addition to addressing it in jamovi.

## Notable Incidents

Where a security incident is identified that may significantly affect users, we will publish details (impact, affected versions, and remediation) via a GitHub Security Advisory and/or [jamovi.org](https://www.jamovi.org).
